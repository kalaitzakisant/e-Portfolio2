Secure Software Development

The learning outcomes of this module were to:
1. Identify and manage security risks as part of a software development project
2. Critically analyse development problems and determine appropriate methodologies, tools and techniques (including program design and development) to solve them.
3. Design and develop/adapt computer programs and to produce a solution that meets the design brief and critically evaluate solutions that are produced.
4. Systematically develop and implement the skills required to be effective member of a development team in a virtual professional environment, adopting real-life perspectives on team roles and organisation.

Unit 1: Introduction to Secure Software Development

Unit 1 Learning Outcomes:
1. Identify relevant academic literature which discusses the ways in which secure software may be developed using an agile process.
2. Decompose an activity into a set of tasks which can be represented using a flow chart from the UML portfolio of modelling techniques.
3. Describe the importance of creating a culture of risk awareness in an organisation, in addition to the creation of secure software.

Discussion 1: UML FlowChart

Unit 2: UML Modelling to Support Secure System Planning.

Unit 2 Learning Outcomes:
1. Research the academic literature on software development which follows an agile process and prioritises security.
2. Break down the steps involved in a process so that they may be represented as a flow chart.
3. Reading areas of an ISO/IEC Standard on security and becoming familiar with the language and concepts.

Unit 3: Programming Languages: History, Concepts & Design

Unit 3 Learning Outcomes:
1. Describe some key milestones in the development of programming languages.
2. Outline some of the key paradigms that define the different types of languages.
3. Explain the key concepts that determine the operation of programming languages.
4. Discuss key programming challenges and recommended best practices.
5. Explain what design patterns are and when to use them.

Unit 4 Learning Outcomes:

2. Describe how and when to use recursion in solutions.
3. Discuss the security implications of both approaches.

Unit 5: An Introduction to Testing

Unit 5 Learning Outcomes:
1. Describe the key terms which are associated with testing software for security.
2. Prepare a list of questions to ask when designing a test plan for secure software.

Unit 6: Using Linters to Support Python Testing

Unit 6 Learning Outcomes:
1. Be familiar with using linters to support quality Python code development.
2. Compare and contrast the outputs from linters to recognise their relevance and applicability in different development and testing scenarios.
3. Develop Python code, which is free of error, consistent in its design, and considered to be high quality according to common Python standards.

Unit 7 Learning Outcomes:
